# 📋 작업 브리프

**원 명령:** 기획서에 맞게 게임을 만들어보자

## 요약
기획 단계에서 개발 단계로 넘어가므로, 현재 완성된 핵심 추론 엔진(Backend)을 기반으로 실제 플레이 가능한 최소 기능 제품(MVP v1)의 범위를 설정해야 합니다. 시장성 및 실현 가능성을 최우선으로 고려하여 다음 단계를 진행합니다.

## 분배
- **💰 Business**: B2P 모델을 유지하며 Steam 출시가 가능한 MVP v1의 구체적인 스코프(Scope)를 정의해주세요. 핵심 기능은 '사건 로딩 → 증거물 분석/파싱 → 모순점 보고서 생성 및 제시' 3단계로 한정하고, 가장 먼저 구현해야 할 핵심 플레이 루프와 필요한 초기 유료 콘텐츠 최소량을 산출합니다.
- **🎨 Designer**: business가 정의한 MVP 스코프에 맞춰 구체적인 게임 메커니즘을 설계해주세요. '전문직 시뮬레이션 추리 RPG'의 핵심 재미를 극대화할 수 있는, 플레이어가 상호작용하는 UI/UX 중심의 인터랙티브 퀘스트 플로우(Flow)와 미니 게임 요소(예: 자료 검색 시스템, 가설 조합 툴 등)를 최소 3가지 이상 구체적으로 제시해주세요.
- **💻 Developer**: business가 정의한 스코프와 game designer가 제안한 메커니즘을 통합하여, 기존의 추론 엔진(Backend)과 연결될 프런트엔드(Frontend/UI) 개발 로드맵을 작성해주세요. 특히 사용자 경험이 가장 중요하므로, '사용자가 데이터를 입력하고 상호작용하는' 핵심 인터페이스 컴포넌트 목록과 API 연동 계획을 우선순위 순으로 정리합니다.
