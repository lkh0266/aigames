# 📋 작업 브리프

**원 명령:** 게임 프로토타입을 만들어 보자

## 요약
게임 프로토타입을 만들기 위해, 핵심 로직(추론 엔진)에 집중하는 최소 기능 제품(MVP) 제작 단계로 전환합니다. 방대한 콘텐츠 구축보다는, 확립된 그래프 모델과 추리 메커니즘이 작동함을 증명하는 데 초점을 맞춥니다.

## 분배
- **💰 Business**: 프로토타입의 범위를 명확히 정의합니다. 현재 목표는 '핵심 논리 증명'입니다. 따라서, 초기 프로토타입은 스토리나 대규모 시스템 구현이 아닌, 단 하나의 샘플 케이스(Case File)에 대한 '데이터 수집 → 관계 매핑 → 모순 발견 → 추론 결과 도출'의 4단계 흐름만 구현하는 것을 목표로 설정하고, 이 범위 제한을 모든 에이전트에게 공지합니다.
- **✍️ Writer**: 프로토타입 테스트용 '샘플 사건 파일(Case File)'의 핵심 데이터를 작성합니다. 최소 3명의 인물 (A, B, C)과 이들이 관련된 5개 이상의 증거물(Evidence Point)을 설정해야 합니다. 각 증거물은 날짜, 금액, 장소 등 논리적 모순이 발생할 수 있는 속성을 포함하도록 구성하고, 이를 개발자에게 전달합니다.
- **💻 Developer**: writer가 제공한 '샘플 사건 파일' 데이터를 입력받아, 핵심 추론 엔진의 최소 기능 제품(MVP)을 구현합니다. 이 MVP는 다음 기능을 포함해야 합니다: 1. 데이터 파싱 및 노드/엣지 생성 기능. 2. 사전에 정의된 규칙 세트(예: 날짜 불일치, 자금 규모 비례 오류)를 기반으로 '모순점 플래그'를 발생시키는 로직 구현. 3. 모순점을 플레이어가 클릭하여 상세한 설명('논리적 증거물')을 얻는 인터페이스의 백엔드 구조 설계.
- **🎨 Designer**: 개발자가 구현할 추론 시스템에 맞춰, '상호작용 가능한 UI 컴포넌트' 디자인 브리프를 작성합니다. 특히, 노드(Node)와 엣지(Edge)가 단순히 그려지는 것을 넘어, 마우스 오버 시 추가 정보가 뜨거나 클릭 시 관련 증거물 목록이 확장되는 등의 인터랙티브 요소를 중심으로 어둡고 고풍스러운 '장부' 스타일의 프로토타입 UI를 설계합니다.
