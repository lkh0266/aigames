# 📑 [디자인 브리프] 모순 보고서 & 가설 생성 인터페이스 (TLA 시스템 결합)

본 브리프는 **[시간(T)-장소(L)-금액(A)]** 데이터 구조를 기반으로, 플레이어가 증거 간의 충돌을 발견하고 새로운 가설을 조립하는 핵심 게임 루프의 비주얼 가이드를 제공합니다.

## 1. 디자인 컨셉: "차가운 자본과 고풍스러운 기록"
- **Visual Mood:** 19세기 금융 장부(Ledger)의 클래식함 + 현대적인 데이터 시각화의 날카로움.
- **Color Palette:**
  - `Primary (Ink)`: `#1A1A1A` (깊은 먹색, 배경 및 기본 텍스트)
  - `Secondary (Paper)`: `#D4C5B3` (낡은 양피지/서류 질감의 베이지)
  - `Accent (Conflict)`: `#B22222` (모순 발생 시 나타나는 강렬한 혈색/인장 레드)
  - `Highlight (Gold)`: `#A68945` (금액 및 중요한 데이터 강조용 골드)
- **Typography:** 
  - 제목/중요 수치: 세리프 계열의 고전적이고 권위 있는 폰트 (예: Playfair Display)
  - 본문/데이터: 가독성이 높은 깔끔한 산세리프 혹은 타자기 스타일 (예: Special Elite / Roboto)

---

## 2. 핵심 UI 레이아웃 설계

### A. [모순 보고서] 화면 (The Conflict Report)
*증거물 사이의 논리적 충돌을 시각적으로 '폭발'시키는 연출이 핵심입니다.*

- **Layout Structure:**
  - **좌측(Evidence List):** 수집된 증거들이 T-L-A 카드로 나열됨. 각 카드는 깔끔한 리스트 형태.
  리듬감 있게 배치된 카드들 사이의 '연결선'이 중요함.
- **Interaction (The Clash):** 
  - 두 개의 증거(예: 시간 $T_1$과 장소 $L_1$)가 물리적으로 양립할 수 없을 때, 두 카드가 서로를 끌어당기며 중앙에서 **'균열(Crack)'**이 발생하는 애니메이션.
  - 충돌 지점에 `Accent Red` 색상의 잉크가 번지는 듯한 시각적 효과 발생.
- **Visual Element:** 
  - 데이터의 불일치 구간을 보여주는 '모순선(Conflict Line)'. 두 카드를 가로지르는 날카로운 대각선이 그어짐.

### B. [가설 생성 UI] 화면 (Hypothesis Builder)
*플레이어가 수집한 단편들을 조합하여 하나의 논리적 흐름을 만드는 작업대입니다.*

- **Layout Structure:**
  - **중앙(The Canvas):** 빈 격자무늬의 장부 스타일 워크스페이스. 플레이어가 카드를 배치하는 공간.
  - **하단/측면(Tool Tray):** 가설 생성을 위해 필요한 '논리적 매개체(Linkers)' 아이콘들.
- **Interaction (The Assembly):**
  - **Drag & Drop:** 증거물 카드(T, L, A)를 드래그하여 빈 슬롯에 배치.
  - **Connection:** 두 카드를 연결할 때 플레이어가 '가설의 선'을 직접 긋는 방식. 
  - 만약 배치가 성공적이면 `Gold` 색상의 빛이 흐르며 연결됨, 모순되면 `Red`하게 진동함.
- **Visual Element:** 
  - 가설 노드(Node) 디자인: 원형이 아닌 육각형 또는 정교한 프레임의 문장(Seal) 형태.

## 3. 데이터 시각화 매커니즘 (T-L-A 시스템 활용)

| 요소 | 디자인 구현 방식 |
| :--- | :--- |
| **Time (T)** | 가로축 혹은 수직 타임라인 흐름으로 표시. 시간 불일치 시 두 선이 엇갈리며 충돌점 발생. |
| **Location (L)** | 지도 좌표 대신, 장소 명칭과 함께 '공간적 거리'를 나타내는 점선(Dashed Line) 활용. |
| **Amount (A)** | 금액의 크기에 따라 카드의 두께나 폰트 무게가 변하는 동적 스케일링 적용. |

## 4. 실행 가능한 UI/UX 가이드라인 요약

1.  **UI Component 'The Ledger Card':** 모든 증거는 종이 질감 배경에 검은색 세리프 폰트로 작성된 카드 형태여야 합니다.
2.  **Interaction 'The Fracture':** 모순을 발견했을 때, 플레이어의 손끝에서부터 시작되는듯한 강력한 진동과 함께 화면 중앙에 붉은 균열이 생기는 연출을 기본으로 합니다.
3.  **UX Flow:** [증거 수집] $\rightarrow$ [모순 탐색(UI 충돌)] $\rightarrow$ [가설 조립(UI 결합)] 순의 사이클로 설계합니다.
