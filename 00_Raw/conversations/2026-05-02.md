## 🎮 Steam 인디 RPG 트렌드 분석 리포트

### 1. 현재 주요 트렌드 (Trend)
현재 Steam 인디 RPG 시장은 '장르의 하이브리드화'와 '전통적 가치의 현대적 재해석'으로 요약됩니다.

*   **장르 융합 (Hybridization):** 순수 RPG보다는 **[RPG + 덱빌딩]**, **[RPG + 경영 시뮬레이션]**, **[RPG + 로그라이트]** 형태가 주류입니다. (예: *Balatro*의 성공 이후 시스템적 중독성을 RPG에 결합하는 추세)
*   **뉴트로 CRPG의 부활:** *Baldur's Gate 3* 이후, 턴제 전투와 깊은 서사, 높은 자유도를 가진 클래식 RPG(CRPG) 스타일의 인디 게임에 대한 수요가 급증했습니다.
*   **코지(Cozy) RPG:** 전투 비중을 낮추고 생활, 관계, 수집에 집중한 '힐링형 RPG'가 특정 니치 층을 넘어 메인스트림으로 진입했습니다.
*   **시스템적 서사 (Systemic Narrative):** 정해진 스크립트가 아니라, 게임 내 시스템(물리, 화학, 사회적 상호작용)이 유저의 행동에 따라 유동적인 이야기를 만들어내는 방식이 각광받고 있습니다.

## 💡 차별화 가능한 니치 시장 포인트 제안

분석 결과, **[깊은 서사]**와 **[시스템적 중독성]**은 존재하지만, 이를 **[특정 전문 영역의 시뮬레이션]**과 결합한 사례는 드뭅니다.

### 🚀 제안: "전문직 시뮬레이션 + 시스템적 추리 RPG"
단순히 몬스터를 잡는 것이 아니라, 특정 전문 분야(예: 고고학자, 법의학자, 도시 계획가 등)의 **'작업 프로세스'를 RPG의 핵심 전투/성장 메커니즘으로 치환**하는 방향입니다.

*   **핵심 컨셉:** `[전문 지식 습득] $\rightarrow$ [현장 조사/분석] $\rightarrow$ [결론 도출 및 세계관 영향]`의 루프.
*   **차별화 포인트 (Niche Point):**
    1.  **전투의 대체:** 칼과 마법 대신 '분석 도구'와 '논리적 가설'을 사용하여 적(혹은 문제)을 해결. (예: 유물을 분석해 고대 언어를 해독하는 과정 자체가 턴제 전투처럼 설계)
    2.  **지식 기반 성장:** 레벨업이 아니라 '논문 작성'이나 '라이브러리 구축'을 통해 캐릭터의 능력이 확장되는 구조.
    3.  **증거 기반 서사:** 유저가 수집한 데이터(물증)에 따라 엔딩과 스토리가 완전히 갈리는 '추리형 로그라이트' 요소 도입.

**결론적으로, "전투 없는 RPG" 혹은 "전투가 분석/추리로 대체된 고지능 RPG" 시장은 현재 Steam에서 수요는 높으나 공급이 부족한 블루오션 영역입니다.**

## ⚔️ 3. 시장 경쟁력 분석 (Competitive Analysis)

### [SWOT 분석]
- **Strength (강점):** '전문직 시뮬레이션'이라는 니치 시장 선점 $\rightarrow$ 대체 불가능한 독점적 경험 제공.
- **Weakness (약점):** 높은 학습 곡선(Learning Curve) $\rightarrow$ 라이트 유저의 이탈 가능성.
- **Opportunity (기회):** 최근 Steam 내 '시스템적 서사' 및 '하이브리드 장르'에 대한 수요 급증.
- **Threat (위협):** 유사한 메커니즘을 가진 대형 인디 스튜디오의 시장 진입.

### [차별화 전략 (Competitive Edge)]
- **"추론의 정량화":** 단순히 텍스트로 추리하는 것이 아니라, 게임 내 시뮬레이션 툴(분석기, 도감 등)을 통해 추론 결과가 수치/증거로 증명되는 시스템 구축 $\rightarrow$ 타 추리 게임과의 기술적 격차 확보.

#### 1. Core Inference System (추론 엔진) 작동 로직 설계

| 구성 요소 | 설명 | 예시 데이터 | 역할 |
| :--- | :--- | :--- | :--- |
| **[Node] 엔티티 (개체)** | 추적의 대상이 되는 모든 실체. | `사람` (엘리아스, 카인), `회사` (에테르노), `금액` (1조 크레딧) | 스토리와 분석의 주축. |
| **[Edge] 관계 (연결고리)** | 노드 간의 연결 및 상호작용 방식. | `거래함`, `만났음`, `소유함`, `날짜_불일치` | 추론의 근거가 되는 핵심 정보. |
| **[Attribute] 속성** | 개별 엔티티나 관계에 부여되는 구체적 수치/정보. | `{날짜: 20XX-MM-DD}`, `{금액: 5억}` | 증거물의 디테일을 제공. |

**B. 추론 프로세스 플로우 (The Deduction Loop)**
1. **[Ingest] 데이터 입력:** 다양한 출처(은행 장부, 이메일)의 데이터를 파싱하여 엔티티와 속성으로 정규화합니다.
2. **[Map] 관계 매핑:** 날짜, 인물 이름, 금액 등의 공통 분모를 찾아 Edge를 생성하고 연결합니다. (예: A의 장부가 B의 출장과 같은 '날짜'라는 Edge 형성)
3. **[Analyze] 모순/이상 탐지:** 시스템이 사전에 정의된 규칙(Rule Set)에 위배되거나, 통계적으로 이상한 패턴을 감지합니다.
    *   **규칙 예시 1 (자금 흐름):** `Edge: [A] $\rightarrow$ [B]`가 존재하는데, `{날짜}`가 다른 모든 거래보다 비정상적일 경우 $\rightarrow$ **'시간차 은폐 의혹'** 플래그 발생.
    *   **규칙 예시 2 (전문성):** `Edge: [직업] $\leftrightarrow$ [자금 흐름]`이 존재하는데, 직업의 전문성과 자금 규모가 비례하지 않을 경우 $\rightarrow$ **'위장 회사 의혹'** 플래그 발생.
4. **[Inference] 추론 결과 출력:** 감지된 모든 모순점과 패턴을 '논리적 증거물(Logical Evidence)'로 플레이어에게 제시합니다.

import datetime
from typing import List, Dict, Any, Tuple

# --- 데이터 구조 정의 ---
class Evidence:
    """개별 증거물 정보를 담는 객체."""
    def __init__(self, evidence_id: str, time: str, location: str, amount: int, detail: str):
        self.evidence_id = evidence_id
        # 시간은 datetime 객체로 변환하여 비교 용이성을 높임
        try:
            self.time = datetime.datetime.strptime(time, "%Y-%m-%d %H:%M")
        except ValueError:
            self.time = None # 파싱 실패 대비
        self.location = location
        self.amount = amount
        self.detail = detail

    def __repr__(self):
        return (f"Evidence(ID={self.evidence_id}, T={self.time.strftime('%H:%M') if self.time else 'N/A'}, "
                f"L='{self.location}', A={self.amount}원)")

class InferenceEngine:
    """
    사건 데이터 파싱, 모순점 플래깅, 분석 보고서 생성을 담당하는 핵심 추론 엔진 MVP.
    """
    def __init__(self):
        self.evidence_list: List[Evidence] = []
        self.contradictions: List[Dict[str, Any]] = []

    # 1. 데이터 파싱 함수 (Data Parsing)
    def parse_evidence(self, raw_data: List[Dict[str, str]]) -> None:
        """
        원시 사건 파일 데이터를 순회하며 Evidence 객체 리스트로 변환합니다.
        raw_data 형식 예시: [{'id': 'E001', 'time': '2026-05-01 10:00', 'location': 'A지점', 'amount': '1000'}, ...]
        """
        print("\n[ENGINE] -> Step 1: 증거물 데이터 파싱 시작...")
        parsed_count = 0
        for item in raw_data:
            try:
                # 문자열 형태의 amount를 정수로 변환하는 과정 포함
                amount = int(item['amount'].replace(',', ''))
                evidence = Evidence(
                    evidence_id=item['id'],
                    time=item['time'],
                    location=item['location'],
                    amount=amount,
                    detail=item['detail']
                )
                self.evidence_list.append(evidence)
                parsed_count += 1
            except ValueError as e:
                print(f"[WARNING] 파싱 실패 (ID: {item.get('id', 'Unknown')}): 데이터 형식 오류 - {e}")

        print(f"[ENGINE] -> Step 1 완료: 총 {parsed_count}개의 증거물이 성공적으로 파싱되었습니다.")
